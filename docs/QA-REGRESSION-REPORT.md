## 1. Область проверки

| Область | Описание |
|--------|----------|
| **Smoke** | Критические сценарии: загрузка приложения, API (слова, статистика, категории), добавление слова (AI), сохранение, повтор в Learn, квиз (все режимы), Cloze, Daily Goal, Слово дня, список/поиск/фильтр, импорт/экспорт, удаление, Story, Chat, Настройки |
| **Регрессия** | Все экраны и переходы: dashboard ↔ add ↔ learn ↔ list ↔ quiz ↔ tutor ↔ story ↔ chat ↔ settings, навбар, модалки (импорт, удаление, детали слова) |
| **Bug hunting** | Обработка null/undefined, ошибки API, граничные случаи (0 слов, 1–3 слова в квизе, cloze без example), целостность данных |

**Окружение:** локальная разработка (`npm run dev`, бэкенд `node server.ts`), SQLite, переменные из `.env` (GEMINI_API_KEY, VITE_API_URL при необходимости).

## 4. Найденные дефекты и исправления

### Критические (исправлены)

| ID | Описание | Исправление |
|----|----------|-------------|
| 1 | **Learn:** при асинхронном изменении списка `dueWords[currentLearnIndex]` мог быть `undefined`, вызов API review с `word.id` приводил к ошибке | В начале `handleReview` добавлена проверка `if (!word) return;` |
| 2 | **Озвучка (playAudio):** при ошибке API `generateSpeech` исключение не перехватывалось, возможен необработанный rejection | Вызов `generateSpeech` обёрнут в try/catch, при ошибке — toast «Ошибка озвучки...» |
| 3 | **Add → Добавить все в словарь:** при сбое одного из POST ответ не проверялся, часть слов могла не сохраниться без уведомления | Проверка `res.ok` по каждому запросу, счётчик сохранённых, при сетевой ошибке — toast с количеством; при успехе — toast «Добавлено слов: N» |
| 4 | **Backend POST /api/words:** допускалось сохранение с пустым `word` | Валидация: при отсутствии или пустой строке `word` — ответ 400 и `{ error: "word is required" }` |

### Незначительные (рекомендации в бэклог)

| ID | Описание | Рекомендация |
|----|----------|--------------|
| 5 | **Stats:** при подстановке ответа API без `todayReviewed` состояние могло бы потерять поле | Нормализовать в `fetchStats`: `setStats({ ...data, todayReviewed: data.todayReviewed ?? 0 })` (сервер уже отдаёт поле) |
| 6 | **Review:** на бэкенде при невалидном `quality` (не 0–3) уровень не меняется, но ошибка не возвращается | Опционально: валидация `quality` и 400 при неверном значении |
| 7 | **Cloze:** при 0 слов с `example` режим недоступен (alert) — ожидаемое поведение | Документировать в подсказке в UI |

## 5. Граничные случаи (проверено)

| Случай | Поведение |
|--------|-----------|
| 0 слов в словаре | Dashboard: пустое состояние, Quiz/Tutor/Story — ограничения или disabled |
| 1–3 слова | Квиз (не Cloze): alert «Нужно минимум 4 слова» |
| Cloze при < 4 слов с example | Аналогичное ограничение |
| Пустой ввод при добавлении слова | Toast «Введите слово для генерации» |
| Импорт невалидного JSON | Toast «Неверный JSON» / «Ошибка импорта» |
| Сбой сети при сохранении/удалении/экспорте | Toast с сообщением об ошибке |
| Последняя карточка в Learn | Переход на dashboard, обновление stats и words |
| Завершение квиза (последний вопрос) | Toast с итоговым счётом, переход на dashboard |

## 6. Рекомендации

1. **Критические исправления** внесены в код (App.tsx, server.ts).
2. **Незначительные** — по желанию внести нормализацию `todayReviewed` в `fetchStats` и валидацию `quality` в POST review.
3. Продолжить развитие: автотесты (e2e/unit) для ключевых сценариев и регрессии.
4. Перед релизом на play2go.cloud повторить smoke с реальным API и окружением.
