# Техническое задание (ТЗ) на доработку веб-приложения "МГИМО AI - Языковая платформа"

## 1. Введение
### 1.1. Общая информация
- **Название проекта**: МГИМО AI - Академическая лексика (MGIMO Language Platform).
- **URL текущей версии**: https://mgimo-ochre.vercel.app/.
- **Заказчик**: Предполагается внутренний проект МГИМО AI Centre (на основе доступной информации о центре ИИ в МГИМО).
- **Исполнитель**: Разработчик или команда, использующая инструменты вроде Cursor AI для генерации и доработки кода.
- **Цель ТЗ**: Описать полный набор требований для доработки прототипа в полноценное веб-приложение для изучения академической лексики с использованием ИИ. Текущая версия - минималистичный прототип на Vercel с нулевыми данными и отсутствующей функциональностью, требующий заполнения контентом, интеграций и улучшений.
- **Версия ТЗ**: 1.0 (от 24 февраля 2026 г.).
- **Объем работ**: Доработка фронтенда, бэкенда, интеграций, тестирование и деплой. Ориентировочная сложность: средняя (2-4 недели для одного разработчика с использованием AI-инструментов вроде Cursor).

### 1.2. Текущий статус
- Приложение - одностраничное (SPA) на базе React.js/Next.js, развернутое на Vercel.
- Видимые элементы: Счетчики (0 слов в базе, 0 на сегодня), разделы "Академический срез" (повторить 0 терминов), "Quiz Mode" (быстрый тест), "Новый термин" (AI-генерация), "Твой словарь" (вкладка "Все").
- Проблемы: Пустая база данных, отсутствие аутентификации, нет реальной AI-интеграции, минимальный UX, отсутствие ошибок в консоли, но полная нефункциональность.
- Технологии (инференс): Front - React/Next.js, CSS (возможно Tailwind), хостинг Vercel. Back - отсутствует или минимальный (serverless functions). Нет видимых API-запросов.

## 2. Цели проекта
- Создать полноценную платформу для студентов МГИМО по изучению академической лексики (термины по международным отношениям, дипломатии, экономике и т.д.).
- Интегрировать ИИ для генерации терминов, примеров предложений, переводов и тестов.
- Обеспечить персонализацию (личный словарь, прогресс).
- Достичь готовности к продакшену: масштабируемость, безопасность, доступность.
- Потенциальная интеграция с системами МГИМО (LMS, аутентификация через университетский аккаунт).

## 3. Требования к системе
### 3.1. Функциональные требования
#### 3.1.1. Фронтенд
- **Интерфейс пользователя**:
  - Адаптивный дизайн (responsive для desktop/mobile, использование media queries).
  - Локализация: Полная поддержка русского и английского (использовать i18n, например react-i18next).
  - Темная/светлая тема (опционально, с сохранением в localStorage).
- **Основные разделы**:
  - **Главная панель**: Отображение счетчиков (слов в базе, на сегодня, терминов для повторения). Динамическая загрузка данных из бэкенда.
  - **Академический срез**: Список терминов для повторения (карточки с словом, переводом, примером). Функция повторения (флэш-карты с интервальным повторением, как Anki).
  - **Quiz Mode**: Быстрый тест (мульти-выбор, ввод текста). Генерация 5-10 вопросов на основе словаря. Отображение результатов (score, ошибки).
  - **Новый термин**: Форма для ручного ввода или AI-генерации. AI должен генерировать: термин, перевод, определение, 2-3 примера предложений (интеграция с OpenAI/Groq или аналогом).
  - **Твой словарь**: Вкладки "Все", "Избранные", "Архив". Поиск, фильтры (по дате, сложности). Экспорт в CSV/PDF.
- **Дополнительные фичи**:
  - Аутентификация: Регистрация/логин (email/password или OAuth via Google/MGIMO).
  - Прогресс-трекинг: Графики (использовать Chart.js), ежедневные цели.
  - Уведомления: Push-уведомления для повторений (Service Worker).
  - Поиск по базе: Глобальный поиск терминов.
- **Технические требования**:
  - Фреймворк: Next.js (для SSR/SSG, API routes).
  - Состояние: Redux или Context API для глобального стора (словарь, пользователь).
  - Стили: Tailwind CSS.
  - Оптимизация: Lazy loading, code splitting.

#### 3.1.2. Бэкенд
- **Архитектура**: Serverless (Vercel Functions или AWS Lambda) для масштабируемости.
- **База данных**: PostgreSQL или MongoDB (интеграция via Supabase/PlanetScale для простоты). Схема:
  - Таблица "users": id, email, password_hash, created_at.
  - Таблица "terms": id, user_id, word, translation, definition, examples (JSON), difficulty, last_reviewed, next_review (для spaced repetition).
  - Таблица "progress": user_id, date, words_learned, quiz_score.
- **API эндпоинты** (REST или GraphQL):
  - GET /api/terms: Получить список терминов (с пагинацией).
  - POST /api/terms: Добавить термин (ручной или AI-generated).
  - PUT /api/terms/:id: Обновить (review, mark as learned).
  - GET /api/quiz: Сгенерировать тест.
  - POST /api/auth: Логин/регистрация.
- **AI-интеграция**: Вызов OpenAI API (GPT-4o или аналог) для генерации. Prompt: "Generate academic term in international relations: word, translation to Russian, definition, 3 examples."
- **Хранение данных**: Обеспечить приватность (GDPR-compliant).

#### 3.1.3. Интеграции
- **AI**: OpenAI/Gemini API (ключ в env variables).
- **Аутентификация**: NextAuth.js или Firebase Auth.
- **Аналитика**: Google Analytics или Vercel Analytics.
- **Хранение файлов**: Если нужны изображения/аудио для терминов - Vercel Blob или S3.
- **Внешние сервисы**: Интеграция с MGIMO API (если доступно, для импорта терминов из курсов).

### 3.2. Нефункциональные требования
- **Производительность**: Загрузка страницы < 2 сек, API response < 500ms. Использовать caching (Redis if needed).
- **Безопасность**:
  - HTTPS, CORS, rate-limiting.
  - Защита от XSS, CSRF (built-in in Next.js).
  - Хранение паролей: bcrypt.
  - Prompt injection protection для AI.
- **Доступность**: WCAG 2.1 (ARIA attributes, keyboard navigation).
- **Масштабируемость**: Поддержка 1000+ пользователей (serverless auto-scale).
- **Совместимость**: Браузеры: Chrome, Firefox, Safari (latest +2 versions).
- **Мобильность**: PWA (manifest, offline via service worker).

## 4. Тестирование
- **Unit tests**: Компоненты фронтенда (Jest + React Testing Library).
- **Integration tests**: API эндпоинты (Supertest).
- **E2E tests**: Cypress или Playwright (тесты на логин, генерацию, quiz).
- **Load testing**: Artillery для симуляции трафика.
- **Security audit**: Snyk или OWASP ZAP.
- **Тестовые данные**: Seed DB с 100+ терминами (из открытых источников по дипломатии).
