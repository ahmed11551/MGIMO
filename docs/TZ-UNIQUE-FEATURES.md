## 1. Анализ рынка

### 1.1 Конкуренты и их слабости

| Приложение | Сильные стороны | Слабые стороны / пробелы |
|------------|-----------------|--------------------------|
| **Anki** | FSRS, мощный SRS | Сложный UI, нет AI, нет геймификации |
| **Quizlet** | Геймификация, Match | Нет академической специализации, слабый SRS |
| **Duolingo** | Привычка, мотивация | Не для словаря, пассивное запоминание |
| **Word of the Day** | Ежедневная привычка | Слова общие, не персонализированы |
| **IELTS/TOEFL apps** | AWL, контекст | Нет AI, нет Telegram, статичный контент |
| **Telegram vocab bots** | Удобство в мессенджере | Нет SRS, нет AI, примитивный функционал |

### 1.2 Тренды 2024–2025 (исследования)

- **AI + контекст:** MIT: персонализация примеров через AI повышает мотивацию
- **Sentence-based SRS:** AllAI — предложения с несколькими словами для повторения → 4× быстрее
- **Cloze deletion:** активное вспоминание (вписать пропуск) эффективнее multiple choice
- **FSRS:** 30% эффективнее SM-2, адаптация под пользователя
- **Heat map прогресса:** визуализация активности (GitHub-style) повышает retention

### 1.3 Уникальная ниша МГИМО AI

- **Академическая лексика** (дипломатия, право, экономика) — узкая специализация
- **Telegram Mini App** — учёба в мессенджере, без установки
- **AI + SRS** — генерация контента + умные повторения
- **Бот + приложение** — слова в чате, напоминания, слово дня

## 2. Уникальные фичи (приоритет)

### 2.1 Слово дня (из своего словаря) — P0

**Проблема:** Word of the Day приложения дают общие слова. Наши пользователи учат академическую лексику.

**Решение:** «Слово дня» — случайное слово **из словаря пользователя** (приоритет: due для повторения или новые).

**Реализация:**
- Виджет на Dashboard: карточка «Слово дня» с одним словом
- Логика: `dueWords[0]` или `words[random]` если due пусто
- Кнопка «Повторить» → переход в Learn
- Бот: команда `/daily` или рассылка в 9:00 (для /remind)

**Уникальность:** Персонализированное слово дня из своей базы.

---

### 2.2 Cloze Mode (заполни пропуск) — P0

**Проблема:** Multiple choice — пассивное узнавание. Cloze — активное вспоминание.

**Решение:** Режим квиза: показывается предложение с пропуском `_____`, нужно выбрать правильное слово из 4 вариантов (или вписать).

**Реализация:**
- Новый режим в Quiz: «Cloze» (пропуск в предложении)
- Бэкенд или фронт: подставляем `_____` вместо слова в `example`
- 4 варианта: правильное + 3 случайных из других слов
- Сохранение в SRS при правильном ответе

**Уникальность:** Контекстное активное вспоминание в академическом формате.

---

### 2.3 Ежедневная цель (Daily Goal) — P0

**Проблема:** Пользователи не знают, «сколько достаточно». Duolingo решает это XP.

**Решение:** Цель «Повторить N слов сегодня» с прогресс-баром.

**Реализация:**
- Настройка: цель по умолчанию 5 слов (или 10, 15)
- Stats: `todayReviewed` — сколько слов повторено сегодня
- Dashboard: прогресс-бар «3/5 слов сегодня»
- При достижении: анимация успеха, toast «Цель достигнута!»
- Хранение: `stats` таблица или localStorage `mgimo-daily-goal`, `mgimo-today-count`

**Уникальность:** Простая, понятная метрика без сложной системы XP.

---

### 2.4 Heat Map активности — P1

**Проблема:** Streak есть, но нет визуализации истории.

**Решение:** GitHub-style календарь: закрашенные дни = активность.

**Реализация:**
- Таблица `stats`: `date`, `words_learned` (уже есть)
- Компонент: сетка 7×12 (недели × месяцы), цвет по интенсивности
- Settings или Dashboard: «Ваша активность за 90 дней»
- Цвета: 0 слов = серый, 1–4 = светло-зелёный, 5+ = тёмно-зелёный

**Уникальность:** Визуальная мотивация, редкая в vocab-приложениях.

---

### 2.5 Режим «Ввод с клавиатуры» — P1

**Проблема:** Выбор из 4 вариантов — легко угадать. Ввод — честная проверка.

**Решение:** Режим Quiz: показать перевод, пользователь вводит слово (с проверкой на опечатки).

**Реализация:**
- Новый quiz mode: `translation-to-typing`
- Input вместо кнопок
- Сравнение: `userInput.toLowerCase().trim()` vs `word.word.toLowerCase()`
- Допуск: Levenshtein distance ≤ 1 или игнор лишних пробелов
- Показ правильного ответа при ошибке

**Уникальность:** Активное вспоминание через ввод, как в Anki type-in.

---

### 2.6 Бот: Слово дня по расписанию — P1

**Проблема:** Пользователь забывает зайти в приложение.

**Решение:** Бот присылает «Слово дня» в 9:00 (или настраиваемое время) тем, кто включил /remind.

**Реализация:**
- Расширить `sendScheduledReminders`: кроме «Пора повторить» — присылать 1 слово
- API: `GET /api/words/random?count=1` (уже есть)
- Формат: как в /word
- Опция: `/remind` включает и напоминания, и слово дня

**Уникальность:** Слово дня в Telegram от своего словаря.

### 2.7 Sentence SRS (AI-предложение с несколькими словами) — P2

**Проблема:** Слова повторяются по одному. Исследование: предложения с 2–3 словами → 4× быстрее.

**Решение:** AI генерирует предложение, в котором встречаются 2–3 слова из due. Пользователь читает, потом карточки по словам.

**Реализация:**
- Новый endpoint: `POST /api/ai/sentence-review` — на вход `wordIds[]`, на выход `{ sentence }`
- Gemini prompt: «Составь одно естественное предложение на английском, используя все слова: X, Y, Z»
- Экран: показать предложение → «Проверить» → показать карточки слов
- Опционально: интеграция в Learn flow

**Уникальность:** Контекстное повторение нескольких слов в одном предложении.

---

### 2.8 Коллокации (AI) — P2

**Проблема:** Слово без типичных сочетаний — неполная картина.

**Решение:** При просмотре слова — блок «Частые сочетания»: mitigate risk, bilateral agreement.

**Реализация:**
- `POST /api/ai/collocations` — `{ word }` → `{ collocations: string[] }`
- Показ в модалке слова / на карточке
- Кэш в БД (опционально)

**Уникальность:** Академические коллокации через AI.
